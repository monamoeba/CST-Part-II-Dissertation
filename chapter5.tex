\subsection{Reflections}


The difficulty of the project stemmed from topic choice of compiling color codes, which currently remains a niche and under-researched area. Scarcity of prior resources and literature introduced a steep learning curve during the preparation stage and difficulties in the implementation phase, particularly when handling undocumented edge cases or complex error mechanisms in the code or compilation pipeline.
However, this allowed the scope of the project to become highly research-based.

The choice to work with an existing codebase provided a chance to develop crucial skills such as navigating and understanding another programmer's code and writing modular programs that can seamlessly integrate into the pipeline. This closely reflects the day-to-day experience of a software engineer, where often, code must be written in a collaborative way to safely modify existing repositories.
